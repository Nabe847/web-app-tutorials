## はじめに

「プログラミングって難しそう...」「本を読んでもよく分からない...」そんな不安を抱えている方も多いと思います。
このチュートリアルでは、そんな不安を解消するために、できるだけ分かりやすく、一つずつ丁寧に説明していきます。

最初は分からないことがたくさんあっても大丈夫です。一緒に少しずつ学んでいきましょう！

## このステップで学ぶこと

- Next.js プロジェクトの作成
- 開発環境の起動
- プロジェクト構造の理解

### 用語の説明

- **ターミナル（コマンドライン）**: パソコンに命令を送るための画面です。Windows では「コマンドプロンプト」や「PowerShell」、Mac では「ターミナル」と呼ばれます。
- **コマンド**: パソコンに実行してほしい命令のことです。例えば「フォルダを作る」「ファイルを開く」などの命令を文字で入力します。

### 前提知識

- HTML、CSS、JavaScript の基礎
- コマンドライン(ターミナル)の基本的な使い方

### 事前準備

Node.js がインストールされていることを確認してください。  
以下のコマンドを実行してください:

```bash
node --version
npm --version
```

以下のようにバージョンが表示されれば OK です:

```
v22.16.0
10.9.2
```

(環境によってバージョン番号が異なる可能性がありますが問題ありません)

Node.js がインストールされていない場合は、[公式サイト](https://nodejs.org/)からダウンロードしてください（LTS 版を推奨）。

### Next.js プロジェクトの作成

ターミナルで以下のコマンドを実行してください:

```bash
npx create-next-app@latest task-manager
```

いくつか質問が表示されます:

```
Would you like to use TypeScript? No / Yes
Would you like to use ESLint? No / Yes
Would you like to use Tailwind CSS? No / Yes
Would you like your code inside a `src/` directory? No / Yes
Would you like to use App Router? (recommended) No / Yes
Would you like to use Turbopack for `next dev`?  No / Yes
Would you like to customize the import alias (`@/*` by default)? No / Yes
```

以下のように回答してください:

```
Would you like to use TypeScript? No / Yes
→ **No** を選択
  理由: TypeScript は高度な機能なので、最初はシンプルな JavaScript で覚えましょう

Would you like to use ESLint? No / Yes
→ **No** を選択
  理由: フレームワークの使い方の理解に集中するため

Would you like to use Tailwind CSS? No / Yes
→ **No** を選択
  理由: CSSの基本から学習するため

Would you like to use `src/` directory? No / Yes
→ **Yes** を選択

Would you like to use App Router? (recommended) No / Yes
→ **Yes** を選択
  理由: 最新の推奨方式のため

Would you like to use Turbopack for `next dev`?  No / Yes
→ **No** を選択
  理由: Next.js の学習に集中するため

Would you like to customize the default import alias (@/*)? No / Yes
→ **No** を選択
  理由: デフォルトのままで十分
```

セットアップが完了したら、プロジェクトフォルダに移動して開発サーバーを起動します:

```bash
cd task-manager
npm run dev
```

以下のような出力が表示されれば成功です:

```
> task-manager@0.1.0 dev
> next dev

   ▲ Next.js 15.3.2
   - Local:        http://localhost:3000
   - Environments: .env.local

 ✓ Starting...
 ✓ Ready in 2.1s
```

ブラウザで `http://localhost:3000` を開くと、Next.js の初期画面が表示されます。

### プロジェクト構造の理解

作成されたプロジェクトの構造:

```
task-manager/
├── app/
│   ├── page.js          # トップページ（/ にアクセスした時）
│   ├── layout.js        # 全ページ共通のレイアウト
│   └── globals.css      # 全体のスタイル
├── public/              # 画像などの静的ファイル
├── package.json         # プロジェクト設定
└── next.config.js       # Next.js の設定
```

**重要なポイント:**

- `app/` フォルダ内のファイル名が URL になります
- `page.js` は各ページのメインコンテンツです
- `layout.js` は全ページで共通するレイアウト（ヘッダーやフッターなど）を定義します

### よくある問題と解決方法

**コマンドを打ち間違えた場合**

- 間違えたコマンドを入力した場合は、`Ctrl + C`（Mac の場合は `Cmd + C`）でキャンセルできます
- もう一度最初から入力し直してください

**ポートが使用中の場合**

- 別のアプリケーションが 3000 番ポートを使用している場合、自動的に 3001 番などに変更されます
- ターミナルに表示される URL を確認してください
- 例：`http://localhost:3001` のように表示された場合は、その URL を開いてください

**コマンドが見つからない場合**

- エラーメッセージ例：`'npx' は内部コマンドまたは外部コマンドとして認識されていません`
- 解決方法：
  1. Node.js が正しくインストールされているか確認
  2. パソコンを再起動してみる
  3. Node.js を再インストールする

**権限エラーが発生する場合**

- エラーメッセージ例：`Permission denied` や `アクセスが拒否されました`
- 解決方法：
  - Mac: Node.js を再インストールする（`sudo` は使わない）
  - Windows: 管理者として実行する必要がある場合があります

**フォルダ名やパスに日本語が含まれている場合**

- エラーが発生する可能性があります
- フォルダ名は英語（アルファベット）のみを使用することをお勧めします
- 例：`タスク管理` → `task-manager`

### 💡 ヒント

**開発サーバーの停止方法:**

- **Windows**: `Ctrl + C` を押す
- **Mac**: `Cmd + C` または `Ctrl + C` を押す
- 押した後、「終了しますか？」と聞かれたら `Y` を押して Enter キーを押してください

**その他のポイント:**

- コマンドを間違えても大丈夫！何度でもやり直せます
- わからない用語があっても、今後のチュートリアルで詳しく説明します
- ファイルを保存すると、ブラウザに表示しているページが自動的に更新されます（ホットリロード機能）
- 分からないことがあったら、焦らずに一つずつ確認していきましょう

### 🎯 理解度チェッククイズ

以下の質問に答えて、学習内容を確認しましょう！

1. **Next.js プロジェクトを作成するコマンドは？**

   - A: `npm create-next-app`
   - B: `npx create-next-app`
   - C: `next create-app`
   - D: `create-next-app`

   <details>
   <summary>答えを見る</summary>

   **答え: B: `npx create-next-app`**

   - `npx` は Node.js のパッケージを実行するためのコマンドです
   - `create-next-app` は Next.js プロジェクトを作成するためのツールです
   </details>

2. **開発サーバーを起動するコマンドは？**

   - A: `npm start`
   - B: `npm run dev`
   - C: `next dev`
   - D: `npm dev`

   <details>
   <summary>答えを見る</summary>

   **答え: B: `npm run dev`**

   - `npm run` は package.json に定義されたスクリプトを実行するコマンドです
   - `dev` は開発用のサーバーを起動するスクリプト名です
   </details>

3. **開発サーバーが起動したら、ブラウザで開く URL は？**

   - A: `http://localhost:3000`
   - B: `http://localhost:8080`
   - C: `http://localhost:4321`
   - D: `http://localhost:5000`

   <details>
   <summary>答えを見る</summary>

   **答え: A: `http://localhost:3000`**

   - `localhost` は自分のパソコンを指します
   - `3000` は Next.js のデフォルトのポート番号です
   </details>

4. **プロジェクトのメインページのファイル名は？**

   - A: `index.js`
   - B: `main.js`
   - C: `page.js`
   - D: `app.js`

   <details>
   <summary>答えを見る</summary>

   **答え: C: `page.js`**

   - Next.js の App Router では、`page.js` が各ページのメインコンテンツを定義します
   - `app` フォルダ内の `page.js` がトップページになります
   </details>

5. **開発サーバーを停止するキーは？**

   - A: `Ctrl + X`
   - B: `Ctrl + C`
   - C: `Ctrl + Z`
   - D: `Ctrl + D`

   <details>
   <summary>答えを見る</summary>

   **答え: B: `Ctrl + C`**

   - Mac の場合は `Cmd + C` も使えます
   - このコマンドで実行中のプログラムを停止できます
   </details>
