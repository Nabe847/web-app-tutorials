## このステップについて

このステップでは、作成したアプリケーションが正しく動作するか確認します。動作確認は、プログラムが期待通りに動くかを確認する重要な作業です。

## 動作確認

### 1. サーバーの起動

まず、バックエンドとフロントエンドの両方のサーバーを起動します：

1. バックエンドサーバーを起動：
```bash
cd todo-manager-backend
uvicorn app.main:app --reload
```

2. フロントエンドサーバーを起動：
```bash
cd todo-manager-frontend
npm run dev
```

### 2. 機能テスト

ブラウザで `http://localhost:3000` を開き、以下の機能を順番にテストします：

1. **Todoの作成**
   - テキストボックスに「買い物に行く」と入力
   - 「追加」ボタンをクリック
   - 新しいTodoがリストに表示されることを確認
   - テキストボックスが空になることを確認

2. **Todoの完了状態の切り替え**
   - 作成したTodoのチェックボックスをクリック
   - Todoのテキストに取り消し線が表示されることを確認
   - 再度チェックボックスをクリック
   - 取り消し線が消えることを確認

3. **Todoの削除**
   - Todoの「削除」ボタンをクリック
   - Todoがリストから消えることを確認

### 3. エラーハンドリングの確認

エラーハンドリングとは、プログラムが予期せぬ状況に対応できるかを確認することです：

1. **バックエンドサーバーを停止**
   - ターミナルで `Ctrl+C` を押してサーバーを停止
   - フロントエンドでTodoの操作を試みる
   - エラーメッセージが表示されることを確認
   - エラーメッセージの内容を確認

2. **バックエンドサーバーを再起動**
   - サーバーを再起動
   - フロントエンドの操作が正常に動作することを確認
   - データが正しく表示されることを確認

## よくある問題と解決方法

**Todoが表示されない場合**

- 解決方法：
  1. バックエンドサーバーが起動しているか確認
  2. ブラウザの開発者ツール（F12）を開く
  3. 「Console」タブでエラーメッセージを確認
  4. 「Network」タブでAPIリクエストの状態を確認
  5. CORSの設定が正しいか確認

**Todoの操作が失敗する場合**

- 解決方法：
  1. バックエンドサーバーが起動しているか確認
  2. ブラウザの開発者ツール（F12）を開く
  3. 「Console」タブでエラーメッセージを確認
  4. 「Network」タブでAPIリクエストの内容を確認
  5. リクエストのURLとパラメータが正しいか確認
