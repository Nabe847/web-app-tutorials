> 💡 **DELETE**: リソースの削除に使われるHTTPメソッドです。データベースからレコードを削除する際に使用します。

## Todo削除機能の作成

`app/main.py` に以下のコードを追加して、Todoの削除機能を作成します:

```python
# app/main.py

# Todoの削除
@app.delete("/todos/{todo_id}")
def delete_todo(todo_id: int, db: Session = Depends(get_db)):
    # 指定されたIDのTodoを取得
    db_todo = db.query(models.Todo).filter(models.Todo.id == todo_id).first()
    
    # Todoが存在しない場合は404エラーを返す
    if db_todo is None:
        raise HTTPException(status_code=404, detail="Todo not found")
    
    # Todoを削除
    db.delete(db_todo)
    
    # データベースに変更を保存
    db.commit()
    
    return {"message": "Todo deleted successfully"}
```

## コードの解説

```python
@app.delete("/todos/{todo_id}")
def delete_todo(todo_id: int, db: Session = Depends(get_db)):
    db_todo = db.query(models.Todo).filter(models.Todo.id == todo_id).first()
    if db_todo is None:
        raise HTTPException(status_code=404, detail="Todo not found")
    db.delete(db_todo)
    db.commit()
    return {"message": "Todo deleted successfully"}
```

- `@app.delete("/todos/{todo_id}")`: DELETEリクエストを処理するエンドポイントを定義します
- `todo_id: int`: パスパラメータを整数型として受け取ります
- `db.query(models.Todo)`: Todoテーブルに対するクエリを作成します
- `filter(models.Todo.id == todo_id)`: 指定されたIDのTodoを検索します
- `db.delete(db_todo)`: Todoを削除します
- `db.commit()`: 変更をデータベースに保存します
- `return {"message": "Todo deleted successfully"}`: 成功メッセージを返します

## よくある問題と解決方法

**404エラーが発生する場合**

- エラーメッセージ例: `{"detail": "Todo not found"}`
- 解決方法:
  1. 指定したIDのTodoが存在するか確認
  2. 正しいIDを指定しているか確認

**データベースエラーが発生する場合**

- エラーメッセージ例: `sqlalchemy.exc.OperationalError`
- 解決方法:
  1. データベースが正しく設定されているか確認
  2. テーブルが存在するか確認

1. **FastAPIでDELETEリクエストを処理するエンドポイントを定義する際の正しい書き方は？**

   - A: `@app.delete("/todos/{todo_id}")`
   - B: `@app.post("/todos/{todo_id}")`
   - C: `@app.get("/todos/{todo_id}")`

<details>
<summary>答えを見る</summary>

   **答え: A: `@app.delete("/todos/{todo_id}")`**

   - DELETEはリソースの削除に使われるHTTPメソッドです
   - FastAPIでは`@app.delete()`デコレータを使ってDELETEリクエストを処理します
   - 参考: [FastAPI公式ドキュメント](https://fastapi.tiangolo.com/tutorial/path-params/)
</details>
