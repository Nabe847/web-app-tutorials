## エラー処理の改善

より良いエラーメッセージを提供するために、カスタム例外クラスを作成します。`app/exceptions.py` を作成し、以下のように編集します:

```python
# app/exceptions.py

from fastapi import HTTPException, status

class TodoNotFoundError(HTTPException):
    def __init__(self):
        super().__init__(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="指定されたTodoが見つかりません"
        )

class InvalidTodoDataError(HTTPException):
    def __init__(self, detail: str):
        super().__init__(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail=detail
        )
```

`app/main.py` を更新し、新しい例外クラスを使います:

```python
# app/main.py

from fastapi import FastAPI, Depends, HTTPException
from sqlalchemy.orm import Session
from . import models, schemas, exceptions
from .database import engine, get_db

# データベースのテーブルを作成
models.Base.metadata.create_all(bind=engine)

app = FastAPI()

# Todoの作成
@app.post("/todos/", response_model=schemas.Todo)
def create_todo(todo: schemas.TodoCreate, db: Session = Depends(get_db)):
    # タイトルが空の場合はエラー
    if not todo.title.strip():
        raise exceptions.InvalidTodoDataError("タイトルは空にできません")
    
    # Todoを作成
    db_todo = models.Todo(title=todo.title)
    db.add(db_todo)
    db.commit()
    db.refresh(db_todo)
    return db_todo

# Todoの一覧取得
@app.get("/todos/", response_model=list[schemas.Todo])
def read_todos(skip: int = 0, limit: int = 100, db: Session = Depends(get_db)):
    todos = db.query(models.Todo).offset(skip).limit(limit).all()
    return todos

# 特定のTodoの取得
@app.get("/todos/{todo_id}", response_model=schemas.Todo)
def read_todo(todo_id: int, db: Session = Depends(get_db)):
    db_todo = db.query(models.Todo).filter(models.Todo.id == todo_id).first()
    if db_todo is None:
        raise exceptions.TodoNotFoundError()
    return db_todo

# Todoの更新
@app.put("/todos/{todo_id}", response_model=schemas.Todo)
def update_todo(todo_id: int, todo: schemas.TodoCreate, db: Session = Depends(get_db)):
    # タイトルが空の場合はエラー
    if not todo.title.strip():
        raise exceptions.InvalidTodoDataError("タイトルは空にできません")
    
    # Todoを取得
    db_todo = db.query(models.Todo).filter(models.Todo.id == todo_id).first()
    if db_todo is None:
        raise exceptions.TodoNotFoundError()
    
    # Todoを更新
    db_todo.title = todo.title
    db.commit()
    db.refresh(db_todo)
    return db_todo

# Todoの完了状態の更新
@app.patch("/todos/{todo_id}/complete", response_model=schemas.Todo)
def complete_todo(todo_id: int, db: Session = Depends(get_db)):
    # Todoを取得
    db_todo = db.query(models.Todo).filter(models.Todo.id == todo_id).first()
    if db_todo is None:
        raise exceptions.TodoNotFoundError()
    
    # Todoを完了状態に更新
    db_todo.completed = True
    db.commit()
    db.refresh(db_todo)
    return db_todo

# Todoの削除
@app.delete("/todos/{todo_id}")
def delete_todo(todo_id: int, db: Session = Depends(get_db)):
    # Todoを取得
    db_todo = db.query(models.Todo).filter(models.Todo.id == todo_id).first()
    if db_todo is None:
        raise exceptions.TodoNotFoundError()
    
    # Todoを削除
    db.delete(db_todo)
    db.commit()
    return {"message": "Todoを削除しました"}
```

## コードの解説

1. **カスタム例外クラス**

   ```python
   class TodoNotFoundError(HTTPException):
       def __init__(self):
           super().__init__(
               status_code=status.HTTP_404_NOT_FOUND,
               detail="指定されたTodoが見つかりません"
           )
   ```

   - `HTTPException`を継承してカスタム例外クラスを作成します
   - `status_code`: HTTPステータスコードを指定します
   - `detail`: エラーメッセージを指定します

2. **エラー処理の実装**

   ```python
   if not todo.title.strip():
       raise exceptions.InvalidTodoDataError("タイトルは空にできません")
   ```

   - バリデーションエラーの処理
   - カスタム例外クラスを使ってエラーを発生させます

## 🚀 動作確認

APIをテストして、エラー処理が正しく動作するか確認しましょう:

1. 存在しないTodoを取得しようとする:
```bash
curl http://localhost:8000/todos/999
```
期待されるレスポンス:
```json

  "detail": "指定されたTodoが見つかりません"
}
```

```json
{
  "detail": "タイトルは空にできません"
}
```

## よくある問題と解決方法

**エラーメッセージが表示されない場合**

- エラーメッセージ例: `Internal Server Error`
- 解決方法:
  1. ログを確認
  2. 例外処理が正しく実装されているか確認
  3. デバッグモードを有効にする

**カスタム例外が機能しない場合**

- エラーメッセージ例: `NameError: name 'exceptions' is not defined`
- 解決方法:
  1. インポート文が正しいか確認
  2. 例外クラスが正しく定義されているか確認
